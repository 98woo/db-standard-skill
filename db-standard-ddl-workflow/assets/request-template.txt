[DB 표준화 업무 테이블 요청 템플릿]

주의:
- 이 파일은 Execution Context가 확정된 뒤 개별 업무 테이블 생성/변경 요청을 정리할 때 사용하는 보조 템플릿이다.
- 최초 context 입력은 `initial-context-new-project.template.md` 또는 `initial-context-existing-project.template.md`를 사용한다.
- 물리 DDL이 CREATE/ALTER/DROP 중 무엇이든 테이블 정의서와 컬럼 정의서 갱신 계획은 필수다.
- 정의서 갱신 계획 없이 물리 DDL만 출력하면 안 된다.

[요청 유형]
요청 유형 : (create_table / alter_add_columns / alter_modify_columns / drop_columns)
실행 요청 : (SQL만 작성 / SQL 작성 후 승인 대기 / 실제 DB 실행)

[공통 업무 정보]
요청 database : (선택. Execution Context와 단일 확정 가능하면 생략 가능)

테이블 종류 : (일반 테이블 / 공간정보 테이블 / 전자정부 프레임워크 / 연계 테이블)
테이블 소유자 :
주제 영역 코드 : (선택. 없으면 테이블 소유자 코드로 namespace 라우팅)

[정의서 갱신 원칙]
- create_table: 테이블 정의서 INSERT + 전체 컬럼 정의서 INSERT + CREATE TABLE 계열 DDL을 함께 생성한다.
- alter_add_columns: 테이블 정의서 row 존재 확인 + 신규 컬럼 정의서 INSERT + ALTER TABLE ADD COLUMN 계열 DDL을 함께 생성한다.
- alter_modify_columns: 컬럼 정의서 UPDATE + ALTER COLUMN 계열 DDL을 함께 생성한다.
- drop_columns: 컬럼 정의서 삭제/비활성화/이력 처리 정책을 확정한 뒤 DROP COLUMN 계열 DDL을 생성한다.
