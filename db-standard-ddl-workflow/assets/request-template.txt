[DB 표준화 작업 시작 템플릿]

주의:
- 이 파일은 개별 테이블 생성/변경 요청을 정리할 때 사용하는 보조 템플릿이다.
- 최초 Execution Context 입력은 아래 Markdown 템플릿 중 하나를 사용한다.
  - initial-context-new-project.template.md
  - initial-context-existing-project.template.md

작업 유형 :
- 신규 프로젝트 표준화 작업 시작
또는
- 기존 프로젝트 표준화 작업 이어서 진행

[공통 입력]
프로젝트 ID :
프로젝트 명 :
DBMS 종류 : (postgresql / oracle / mysql / mariadb / sqlserver)
DBMS 버전 : (모르면 비워둘 수 있음)
DBMS 접속 대상 : (Oracle service/PDB, SQL Server instance 등. 접속 도구가 별도 관리하면 비워둘 수 있음)
DBMS profile : auto (권장)
실행 요청 :
- SQL만 작성
또는
- SQL 작성 후 승인 대기
또는
- 실제 DB 실행

[신규 프로젝트일 때 추가 입력]
물리 대상 데이터베이스/접속 대상 :
물리 대상 namespace 라우팅 :
- namespace명(주제영역코드 또는 소유자코드)
- namespace명(주제영역코드1, 주제영역코드2)

참고: namespace 의미는 DBMS별로 다름
- PostgreSQL: schema
- Oracle: schema/user/owner
- MySQL/MariaDB: database
- SQL Server: schema

프로젝트 산출물 namespace : db_standard (고정)

컬럼 정의서 한글 테이블명 :
컬럼 정의서 한글 컬럼 목록 :
- 한글 컬럼명

참고 :
- 신규 프로젝트 정의서 테이블 생성은 표준화 작업이 아니다.
- 에이전트가 한글 컬럼명을 자동 snake_case 물리 컬럼명으로 변환한다.
- 정의서 테이블 컬럼 타입은 모두 text로 생성한다.
- 한글 테이블명 / 컬럼명은 comment로 추가한다.
